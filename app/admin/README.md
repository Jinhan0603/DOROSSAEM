# 🤖 DORO Admin - 강사 관리 & 일정 배정 시스템

DORO 강사 관리, 강의 평가, 일정 배정을 위한 웹 기반 관리자 대시보드입니다.

## 시작하기

### 1. 실행
브라우저에서 `admin/index.html` 파일을 직접 열거나:
```
file:///C:/Users/User/OneDrive/문서/Jindex/doro-website/admin/index.html
```

또는 HTTP 서버로 실행:
```bash
cd doro-website
python -m http.server 8080
# http://localhost:8080/admin/ 접속
```

### 2. 초기 설정
1. 사이드바 하단 ⚙️ **설정** 클릭
2. **데모 데이터 생성** 클릭 → 테스트용 강사 5명, 강의, 공고 데이터 생성
3. **대시보드**로 이동하여 전체 현황 확인

---

## 📋 기능별 사용법

### 대시보드
메인 화면에서 4개 처리 큐를 한눈에 확인합니다.

| 큐 | 의미 | 클릭 시 |
|---|---|---|
| 🔴 미완료 강의 | 오늘까지 완료 처리 안 된 강의 | 강의 관리로 이동 |
| 🟡 평가 미회수 | 완료됐지만 평가 안 된 강의 | 강의 평가로 이동 |
| 🔵 기여 승인대기 | 내부기여 승인 대기 건 | 승인 대기로 이동 |
| 🟢 Incident | 가산/감점 기록 | 가산/감점으로 이동 |

---

### 강의 관리 (Lecture Master)
**기존 단건 강의를 직접 관리**하는 메뉴입니다.

1. **새 강의** 버튼 → 날짜, 기관, 강의명, 주/보조강사 선택
2. 예정된 강의 행의 **✅ 완료** 버튼 클릭
3. → Activity_Log 자동 생성 + 강사 점수 자동 재계산

---

### 📢 강의 공고 (신규)
**기관에서 요청한 강의를 공고로 등록**하는 메뉴입니다.

#### 공고 등록
1. **공고 등록** 버튼 클릭
2. 기관, 콘텐츠, 대상, 인원, 지역, 유형, 중요도 입력
3. 강사비 단가 설정 (주강사 2.5만원/h, 보조 1.2만원/h 기본)
4. 기관 지급 단가 입력 (차액이 있으면 페이백 금액 자동 계산)
5. 전문성 태그 입력 (예: `코딩,로봇,AI`)

#### 회차 생성
1. 초안 상태의 공고에서 **회차생성** 클릭
2. 회차 수, 각 회차별 날짜/시간 설정
3. **생성 및 공고 오픈** → 공고가 모집중으로 변경

---

### 📅 일정 배정 (신규 - 핵심)
**추천 엔진 기반으로 강사를 배정**하는 메뉴입니다.

#### 배정 흐름
```
추천 후보 보기 → 홀드(24시간) → 확정 → 수업 완료
```

1. 배정할 회차의 **🪄 추천 후보 보기** 클릭
2. 시스템이 자동으로 3가지 추천안 제시:

| 추천안 | 설명 |
|--------|------|
| 🟢 안정형 | 적합도 점수 최고 (성공 확률 최고) |
| 🔵 수익형 | 최소 필요 등급 배정 (과티어 방지) |
| 🟡 백업형 | 가용시간 확실한 후보 |

3. **이 조합 홀드** 또는 개별 강사 **홀드/즉시확정** 클릭
4. 홀드 후 **확정** 버튼으로 최종 배정
5. 수업 종료 후 **수업 완료** → 결과 원장 + Activity_Log 자동 생성

#### 추천 엔진 구조
```
1차 하드 필터 (탈락 시 즉시 제외)
  - 비활성/차단 상태
  - 시간 겹침 (기존 배정)
  - 가용시간 불일치
  - 등급 권한 불일치

2차 적합도 점수 (100점)
  - 역할/등급 적합도: 30점
  - 전문성 매칭: 25점
  - 시간/지역 적합도: 20점
  - 동일 콘텐츠/기관 경험: 15점
  - 신뢰도: 10점

3차 보정
  - 반복 과정 동일 강사: +10점
  - 과부하 (최근 7일 3건+): -15점
  - 과티어 (일반 수업에 Master): -5점
```

---

### 🕐 가용시간 관리 (신규)
**강사별 가용 요일/시간을 관리**합니다.

1. 강사 선택
2. 월~일 × 오전/오후/저녁 체크박스 편집
3. 활동 지역, 이동수단 설정
4. **저장** 클릭
5. → 일정 배정 시 충돌 검사에 자동 반영

---

### 📊 결과 원장 (신규)
**기존 수기 엑셀을 대체**하는 강의 결과 기록입니다.

- 수업 완료 시 자동 생성됨 (수기 입력 불필요)
- 날짜, 기관, 콘텐츠, 강사, 시간, 강사비, 페이백 자동 표시
- **CSV 내보내기** → 월간 보고서 작성용

---

### ⭐ 강의 평가
1. **강의 평가** 메뉴 → 평가 대기 목록 확인
2. 각 강사 행의 **✏️ 입력** 클릭
3. 학생/기관/동료 탭 전환 → 만족도 1~5점 입력
4. → 평가 점수 자동 계산 (학생 40% + 기관 40% + 동료 20%)

---

### ⏳ 승인 대기 (내부기여)
1. **기여 등록** → 강사 선택 → 항목 선택 (교안 제작 10p, OT 2p 등)
2. 증빙 URL 첨부
3. 목록에서 **✅ 승인** 또는 **❌ 반려**
4. 승인 시 → Activity_Log 자동 append → 점수 반영

---

### ⚠️ 가산 / 감점
1. **Incident 기록** → 강사 선택 → 사유 선택
   - 가산: 학생 칭찬 +5p, 기관 재요청 +10p, 대규모 행사 +3p
   - 감점: 지각 -10p, 노쇼 -30p (즉시 배정차단)
2. 승인 대기로 이동 → 매니저가 최종 승인

---

### 👤 강사 목록
- 강사 행 클릭 → 프로필 모달
- 4대 점수 바 차트 + 활동 이력 타임라인 확인
- 필터: 전체 / Master / 차단됨

---

### 📈 월간 리포트
- **스냅샷 생성** → 현재 시점의 전 강사 점수 기록
- Master 재산정 근거로 활용

---

## 점수 체계

```
총점 = 경험(20%) + 평가(40%) + 전문성(20%) + 기여(20%) + 가산/감점
```

| 등급 | 조건 | 역할 범위 |
|------|------|----------|
| 👑 Master (≥70) | 총점 70 이상 | DLS/캠프/행사/일반 주강 모두 가능 |
| ⭐ Standard (≥40) | 총점 40~69 | 일반 수업 주강 가능 |
| 🌱 Trainee (<40) | 총점 40 미만 | 보조강사 중심 |

## 강사비 구조

| 역할 | DORO 기준단가 | 기관 지급 시 |
|------|-------------|------------|
| 주강사 | 2.5만원/h | 차액을 DORO에 수수료 페이백 |
| 보조강사 | 1.2만원/h | 차액을 DORO에 수수료 페이백 |

결과 원장에서 기관 지급 단가와 DORO 기준단가의 차액(페이백 금액)이 자동 계산됩니다.

## 데이터 관리

| 기능 | 위치 |
|------|------|
| CSV 전체 내보내기 | 설정 |
| CSV 강사 가져오기 | 강사 목록 |
| 결과 원장 CSV | 결과 원장 |
| 데이터 백업 | 사이드바 하단 |
| 전체 초기화 | 설정 |
| 전체 점수 재계산 | 설정 |

> ⚠️ 현재 데이터는 브라우저 LocalStorage에 저장됩니다. 브라우저 캐시를 지우면 데이터가 삭제되므로, 정기적으로 CSV 백업을 사용하세요.

```
├── scheduling-model.js  # 일정 데이터 모델 + 추천 엔진
└── scheduling-app.js    # 일정 UI (공고, 배정, 가용시간, 원장)
```
