# Simplified DB Model

## 한 줄 요약

`강사 기본 정보`는 가볍게, `활동`은 로그로, `결과`는 역할/등급으로 본다.

## 1. 매니저가 직접 넣는 정보

- 이름
- 최초 기수
- 이번 기수 활동 여부
- 학교
- 학과
- 전공/프로젝트 경험
- 거주지
- 개인정보

## 2. 시스템이 쌓는 정보

- 수업 참여
- OT / 교육 참여
- 제작/운영 알바 참여
- 추천
- 우수 강사 선정
- 패널티
- 피드백

## 3. 매니저가 최종적으로 보는 값

- `보조강사 배정`
- `주강사 배정`
- `DLS 주강사 배정`
- `운영인력 배정`
- `등급 (Tier)`
- `최종 점수`

## 4. 왜 이 구조가 단순한가

- 입력 시트와 판단 시트를 분리했다.
- 점수는 하나만 남겼다.
- 역할 컬럼이 바로 의사결정에 쓰인다.
- 패널티가 별도 로그로 남아 누락이 줄어든다.
